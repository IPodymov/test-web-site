Выполнена автоматизированная проверка сайта по множеству сценариев. В ходе тестирования проверялись: базовые HTTP-статусы, качество SEO, производительность, доступность верстки (a11y) и корректность ссылок.

Основные выявленные недочеты носят технический характер:

- Местами присутствуют переоптимизированные по длине title и meta-теги.
- Часть изображений содержит технические alt-тексты (например, "image" или "brand"), что снижает доступность страницы.
- Отсутствуют базовые семантические landmark-элементы (`<main>`, `<nav>`).

Эти проблемы легко поправимы и их исправление позволит довести качество технической базы до идеала.

## 4. Как справляетесь с жёсткими дедлайнами?

Мой подход при жёстких дедлайнах:

1. **Приоритизация по MoSCoW** — разделяю задачи на Must/Should/Could/Won't. В первую очередь — то, без чего продукт не работает.

2. **Декомпозиция до часовых задач** — разбиваю крупные задачи на куски по 1–4 часа. Это делает прогресс измеримым и снижает тревогу.

3. **Честная коммуникация** — если дедлайн нереалистичен, говорю об этом заранее с конкретными цифрами, а не молчу до последнего.

4. **Технический долг в явном виде** — то, что делаю быстро «наживую», помечаю в коде комментарием `TODO` или создаю тикет. Не делаю вид, что костылей нет.

5. **Не жертвую тестами** — даже под давлением пишу хотя бы smoke-тесты, потому что баги в проде обходятся дороже времени на тест.

## 5. Как развернуть приложение на сервере?

Мой типичный процесс деплоя (на примере Python/Node.js приложения):

### Вариант A — VPS/Dedicated (опыт)

```
1. Провизионирование сервера
   - Ubuntu 22.04, настройка ufw, fail2ban, SSH-ключи
   - Установка зависимостей (nginx, certbot, docker)

2. CI/CD (GitHub Actions или GitLab CI)
   - На push в main: тесты → build → SSH deploy
   - Секреты в GitHub Secrets

3. Docker Compose
   - app (Python/Node) + nginx + postgres/redis
   - .env через docker secrets или Vault

4. Nginx как reverse proxy
   - HTTPS через Let's Encrypt (certbot --nginx)
   - gzip, headers безопасности, rate limiting

5. Запуск + мониторинг
   - systemd или supervisord для auto-restart
   - Логи в /var/log + ротация через logrotate
   - Uptime-мониторинг (UptimeRobot или Prometheus + Grafana)
```

### Вариант B — PaaS (Railway, Render, Fly.io)

- Для MVP и небольших проектов — деплой из Git в один клик
- Меньше DevOps, быстрее старт, дороже при масштабировании

## 6. Хранение медиа-файлов для сайта

Для сайта уровня Studia 54 (много тяжёлых изображений и видео):

### Рекомендуемая архитектура:

```
Загрузка →  Backend API
                ↓
         S3-совместимое хранилище
         (AWS S3 / Yandex Object Storage / Cloudflare R2)
                ↓
          CDN (CloudFront / Cloudflare)
                ↓
          Пользователь (ближайший edge-узел)
```

### Детали:

- **S3 (или аналог)** — дешёвое и надёжное хранилище. Для РФ — Yandex Object Storage или VK Cloud Storage.
- **CDN** — обязателен для медиа, иначе изображения грузятся с одного сервера. Cloudflare бесплатен для базового использования.
- **Обработка изображений на лету** — Cloudflare Images, imgproxy или thumbor: автоматическое ресайзирование, конвертация в WebP.
- **Именование файлов** — хеш содержимого в имени (как на этом сайте: `image_7_3951b87320.webp`) — правильный подход, позволяет бесконечный cache TTL.
- **Видео** — отдельный вопрос; для коротких промо-роликов — Cloudflare Stream или Vimeo Pro с embed.

## 8. Пример кода — тест-скрипт для сайта

Сам по себе этот тестовый набор и является примером кода. Особо горжусь модулем [tests/test_links.py](./tests/test_links.py):

```python
def collect_internal_links(soup, base_url: str) -> list[str]:
    """Извлечь все уникальные внутренние ссылки со страницы."""
    links = set()
    for a_tag in soup.find_all("a", href=True):
        href = a_tag["href"].strip()
        if is_excluded(href):
            continue
        absolute = urljoin(base_url, href)
        parsed = urlparse(absolute)
        if parsed.netloc == urlparse(base_url).netloc:
            # Нормализуем: убираем якори и query-параметры
            clean = parsed._replace(fragment="", query="").geturl()
            links.add(clean)
    return sorted(links)
```

**Почему горжусь:**

- Функция чистая (pure function), без побочных эффектов
- Нормализация URL предотвращает дублирование (`/about` и `/about#team` → одна ссылка)
- `is_excluded()` выделена отдельно — легко расширять список исключений
- Возвращает отсортированный список → детерминированный порядок тестов
