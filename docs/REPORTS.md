# Отчеты и результаты (reports/)

При каждом запуске `run_tests.py` система стирает или перезаписывает предыдущие результаты, сохраняя новые актуальные артефакты в папку `reports/`.

## 1. `REPORT.pdf`

Графический, читаемый отчет. Созданный специально для топ-менеджеров, руководителей разработки (CTO) или HR:

- Содержит оценку качества сайта (Quality Score) в %.
- Общее количество успешных и упавших тестов.
- Подробное описание всех упавших тестов (багов), сгруппированных по критичности (🔴 Высокая критичность, 🟡 Средняя и т.д.).
- Отдельно содержит импортированные ответы на вопросы тестового задания (парсинг `ANSWERS.md` в HTML).

> Механика работы: `pytest-json-report` → `Python` (данные контекста) → `Jinja2` (HTML-шаблон) → `Playwright Headless Chrome` → PDF файл в формате A4 с 15мм отступами.

## 2. `REPORT.md` и `BUGS.md`

Два текстовых отчета в формате Markdown, удобных для чтения через IDE разработчика или загрузки в GitHub/GitLab:

- **`REPORT.md`**: краткая сводка, список пропущенных тестов по категориям, базовое резюме (`Quality Score`).
- **`BUGS.md`**: детальный список ошибок (stack traces), где затронуты сообщения конкретных pytest ассертов для каждого упавшего теста.

## 3. `results.json`

Сырой машиночитаемый отчет, генерируемый библиотекой `pytest-json-report`. Он содержит абсолютную информацию об окружении, времени запуска, node_id каждого теста, traceback исключений и тд. Этот файл служит базой данных, на основе которой строятся все остальные отчеты во время выполнения `run_tests.py`.
