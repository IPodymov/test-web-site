# Архитектура проекта

Этот проект предназначен для автоматизированного тестирования веб-сайтов (на примере studia-54.com).

## Структура тестов (`tests/`)

Все тесты основаны на `pytest` и используют `requests` + `beautifulsoup4` для статического анализа или работы с HTTP. Они разбиты по следующим категориям:
- `test_accessibility.py` — проверка базовой доступности (alt-тексты изображений кнопок, ARIA-метки, HTML-семантики вроде `<main>`).
- `test_functional.py` — функциональные проверки: наличие форм обратной связи, присутствие телефона в шапке/подвале и т.д.
- `test_http_status.py` — проверка кодов ответа `200 OK` ключевых маршрутов, правильность редиректов (например с поддомена `www` на основной домен).
- `test_links.py` — сбор и анализ внутренних/внешних ссылок на отсутствие битых (статус `404 Not Found`).
- `test_performance.py` — сбор базовых метрик: вес изображений, заголовки компрессии (`content-encoding`), время ответа от сервера (TTFB).
- `test_seo.py` — проверка длины тегов `<title>` и `<meta name="description">` на соответствие стандартам, проверка уникальности заголовков `<h1>`.

## Запуск тестов (`run_tests.py`)

Скрипт `run_tests.py` является единой точкой входа. Больше нет необходимости отдельно запускать pytest, затем собирать отчеты.

Внутри он делает следующее:
1. Запускает `pytest` через подпроцесс (subprocess).
2. Требует от `pytest` сгенерировать JSON-отчет в скрытой фоновой структуре (`pytest-json-report`).
3. Берет сырые JSON-данные и парсит их, разбивая на успешные (passed), упавшие (failed) и пропущенные (skipped).
4. Подсчитывает `Quality Score` (% пройденных тестов).
5. Создает из этого объекты контекста для построения `Markdown` (`REPORT.md`, `BUGS.md`) отчетов, а затем транслирует их через `Jinja2` (строковый рендеринг) для `HTML` макета.
6. Вызывает `Playwright` в headless-режиме, скармливает ему HTML и сохраняет в чистый PDF.
